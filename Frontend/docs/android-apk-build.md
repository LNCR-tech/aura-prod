# Android APK Build Guide

This project can now package as a Capacitor Android app without hardcoding the backend host into the source code.

## What the Android shell uses

- Backend base URL comes from env values such as `VITE_NATIVE_API_BASE_URL`
- Capacitor config is generated by [`scripts/write-capacitor-config.mjs`](/d:/Aura/aura-frontend/scripts/write-capacitor-config.mjs)
- Android assets are synced from the Vite production build into [`aura-apk/android`](/d:/Aura - Frontend/aura-frontend/aura-apk/android) with `npx cap sync android`

## Required env

Set these before building:

```env
VITE_USE_MOCK_API=false
VITE_API_BASE_URL=/__backend__
VITE_NATIVE_API_BASE_URL=https://backend-api-production-eeb9.up.railway.app
VITE_BACKEND_PROXY_TARGET=https://backend-api-production-eeb9.up.railway.app
```

Use the backend root URL, not the `/api` suffix.

## Build commands

From [`aura-frontend`](/d:/Aura/aura-frontend):

```powershell
npm install
npm run android:build:debug
npm run android:build:release
```

## Output files

- Debug APK: [`aura-apk/android/app/build/outputs/apk/debug/app-debug.apk`](/d:/Aura - Frontend/aura-frontend/aura-apk/android/app/build/outputs/apk/debug/app-debug.apk)
- Release APK: [`aura-apk/android/app/build/outputs/apk/release/app-release-unsigned.apk`](/d:/Aura - Frontend/aura-frontend/aura-apk/android/app/build/outputs/apk/release/app-release-unsigned.apk)

The release build currently generates an unsigned APK. That is normal until you add your signing key for Play Store or production distribution.

## Runtime permissions

The Android app already declares these permissions in [`AndroidManifest.xml`](/d:/Aura - Frontend/aura-frontend/aura-apk/android/app/src/main/AndroidManifest.xml):

- `INTERNET`
- `ACCESS_NETWORK_STATE`
- `CAMERA`
- `ACCESS_FINE_LOCATION`
- `ACCESS_COARSE_LOCATION`
- `VIBRATE`
- `WAKE_LOCK`

Runtime behavior:

- Location is requested through the shared Capacitor geolocation flow in [`devicePermissions.js`](/d:/Aura/aura-frontend/src/services/devicePermissions.js)
- Camera is requested before starting attendance scanners in [`AttendanceView.vue`](/d:/Aura/aura-frontend/src/views/dashboard/AttendanceView.vue) and [`QuickAttendanceView.vue`](/d:/Aura/aura-frontend/src/views/auth/QuickAttendanceView.vue)
- Network state does not use an Android runtime permission prompt; it is checked via [`useNetworkStatus.js`](/d:/Aura/aura-frontend/src/composables/useNetworkStatus.js) and shown in the offline banner in [`App.vue`](/d:/Aura\aura-frontend\src\App.vue)

## Install on Android phone

For local testing:

```powershell
adb install -r aura-apk\android\app\build\outputs\apk\debug\app-debug.apk
```

Or copy the APK to the phone and install it manually.

## Rebuild after backend change

If the backend URL changes:

1. Update the env file
2. Run `npm run android:build:debug` again

The Capacitor config will regenerate automatically for the new backend host.
