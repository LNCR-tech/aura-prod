# Android APK Build Guide

This project can now package as a Capacitor Android app for pilot testing without hardcoding the backend host into the source code.

## What the Android shell uses

- Backend base URL comes from env values such as `VITE_NATIVE_API_BASE_URL`
- Capacitor config is generated by [`scripts/write-capacitor-config.mjs`](../scripts/write-capacitor-config.mjs)
- Android assets are synced from the Vite production build into [`aura-apk/android`](../aura-apk/android) with `capacitor sync android`

## Required env

Set these before building:

```env
VITE_USE_MOCK_API=false
VITE_API_BASE_URL=/__backend__
VITE_NATIVE_API_BASE_URL=https://your-ngrok-backend.ngrok-free.app
VITE_BACKEND_PROXY_TARGET=https://your-ngrok-backend.ngrok-free.app
VITE_ENABLE_PREVIEW_ROUTES=false
VITE_ENABLE_API_LAB=false
```

Use the backend root URL, not the `/api` suffix. Use an HTTPS ngrok URL for pilot builds.

## Build commands

From [`frontend`](..):

```powershell
npm install
npm run android:build:debug
npm run android:build:release
```

## Output files

- Debug APK: [`aura-apk/android/app/build/outputs/apk/debug/app-debug.apk`](../aura-apk/android/app/build/outputs/apk/debug/app-debug.apk)
- Release APK: [`aura-apk/android/app/build/outputs/apk/release/app-release.apk`](../aura-apk/android/app/build/outputs/apk/release/app-release.apk)

Pilot builds now fall back to the Android debug signing key when no custom keystore is configured. That makes the release APK installable for controlled testing. For production or Play Store distribution, set a real release keystore through Android signing env vars.

## Pilot hardening defaults

- Internal preview routes under `/exposed/*` are disabled in production builds unless `VITE_ENABLE_PREVIEW_ROUTES=true`.
- `ApiLab` is disabled in production builds unless `VITE_ENABLE_API_LAB=true`.
- The Android shell now expects HTTPS backend traffic for pilot builds. Use an ngrok HTTPS tunnel instead of cleartext local IP hosts.

## Runtime permissions

The Android app already declares these permissions in [`AndroidManifest.xml`](../aura-apk/android/app/src/main/AndroidManifest.xml):

- `INTERNET`
- `ACCESS_NETWORK_STATE`
- `CAMERA`
- `ACCESS_FINE_LOCATION`
- `ACCESS_COARSE_LOCATION`
- `POST_NOTIFICATIONS`
- `RECEIVE_BOOT_COMPLETED`
- `VIBRATE`
- `WAKE_LOCK`

Runtime behavior:

- Location is requested through the shared Capacitor geolocation flow in [`devicePermissions.js`](../src/services/devicePermissions.js)
- Camera is requested before starting attendance scanners in [`AttendanceView.vue`](../src/views/dashboard/AttendanceView.vue) and [`QuickAttendanceView.vue`](../src/views/auth/QuickAttendanceView.vue)
- Notification display is requested by the local notification service before showing attendance or event notifications.
- Network state does not use an Android runtime permission prompt; it is checked via [`useNetworkStatus.js`](../src/composables/useNetworkStatus.js) and shown in the offline banner in [`App.vue`](../src/App.vue)
- Storage permissions are intentionally not declared. APK exports write to the app cache through Capacitor Filesystem and then use Android Share/FileProvider, so broad `READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE`, or `READ_MEDIA_*` access is not required.
- Background location is intentionally not declared. Attendance geofencing runs only while the user is actively checking in/out.
- Exact alarm permissions are intentionally not declared. Aura uses immediate local notifications for attendance and foreground/return-to-app event alerts instead of exact scheduled alarms.

## Native event notifications

The APK starts a native-only event notification sync from [`eventNotificationSync.js`](../src/services/eventNotificationSync.js).

- Ongoing events trigger a local notification once per event window.
- Newly loaded events trigger a local notification after the app has already seen the user's existing event list.
- The native app refreshes dashboard events on startup, foreground resume, and a 60-second foreground poll.
- These are local Capacitor notifications. For notifications while the app has never opened or is fully killed, the backend would need server push/FCM.

## Android network config

- `VITE_NATIVE_API_BASE_URL` or `VITE_BACKEND_PROXY_TARGET` controls the backend host used by the APK.
- The generated Capacitor config allows the configured backend hostname plus Railway and ngrok app/dev host patterns.
- The Android network security config now blocks cleartext backend traffic for pilot safety. Use HTTPS ngrok endpoints for Android APK testing.

## Install on Android phone

For local testing:

```powershell
adb install -r aura-apk\android\app\build\outputs\apk\release\app-release.apk
```

Or copy the APK to the phone and install it manually.

## iPhone / iPad PWA

- Build the web app with the same ngrok backend env.
- Serve the built frontend over HTTPS.
- Open the frontend ngrok URL in Safari, then use **Share â†’ Add to Home Screen**.
- `dist/runtime-config.js` now writes both web and native API base URLs when an absolute ngrok backend URL is configured, so the iOS PWA can call the same tunnel directly.

## Rebuild after backend change

If the backend URL changes:

1. Update the env file
2. Run `npm run android:build:release` again

The Capacitor config will regenerate automatically for the new backend host.
