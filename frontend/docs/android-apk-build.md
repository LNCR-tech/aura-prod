# Android APK Build Guide

This project can now package as a Capacitor Android app without hardcoding the backend host into the source code.

## What the Android shell uses

- Backend base URL comes from env values such as `VITE_NATIVE_API_BASE_URL`
- Capacitor config is generated by [`scripts/write-capacitor-config.mjs`](/d:/Aura - Frontend/aura-frontend/scripts/write-capacitor-config.mjs)
- Android assets are synced from the Vite production build into [`aura-apk/android`](/d:/Aura - Frontend/aura-frontend/aura-apk/android) with `npx cap sync android`

## Required env

Set these before building:

```env
VITE_USE_MOCK_API=false
VITE_API_BASE_URL=/__backend__
VITE_NATIVE_API_BASE_URL=https://backend-api-production-eeb9.up.railway.app
VITE_BACKEND_PROXY_TARGET=https://backend-api-production-eeb9.up.railway.app
```

Use the backend root URL, not the `/api` suffix.

## Build commands

From [`aura-frontend`](/d:/Aura - Frontend/aura-frontend):

```powershell
npm install
npm run android:build:debug
npm run android:build:release
```

## Output files

- Debug APK: [`aura-apk/android/app/build/outputs/apk/debug/app-debug.apk`](/d:/Aura - Frontend/aura-frontend/aura-apk/android/app/build/outputs/apk/debug/app-debug.apk)
- Release APK: [`aura-apk/android/app/build/outputs/apk/release/app-release-unsigned.apk`](/d:/Aura - Frontend/aura-frontend/aura-apk/android/app/build/outputs/apk/release/app-release-unsigned.apk)

The release build currently generates an unsigned APK. That is normal until you add your signing key for Play Store or production distribution.

## Runtime permissions

The Android app already declares these permissions in [`AndroidManifest.xml`](/d:/Aura - Frontend/aura-frontend/aura-apk/android/app/src/main/AndroidManifest.xml):

- `INTERNET`
- `ACCESS_NETWORK_STATE`
- `CAMERA`
- `ACCESS_FINE_LOCATION`
- `ACCESS_COARSE_LOCATION`
- `POST_NOTIFICATIONS`
- `RECEIVE_BOOT_COMPLETED`
- `VIBRATE`
- `WAKE_LOCK`

Runtime behavior:

- Location is requested through the shared Capacitor geolocation flow in [`devicePermissions.js`](/d:/Aura - Frontend/aura-frontend/src/services/devicePermissions.js)
- Camera is requested before starting attendance scanners in [`AttendanceView.vue`](/d:/Aura - Frontend/aura-frontend/src/views/dashboard/AttendanceView.vue) and [`QuickAttendanceView.vue`](/d:/Aura - Frontend/aura-frontend/src/views/auth/QuickAttendanceView.vue)
- Notification display is requested by the local notification service before showing attendance or event notifications.
- Network state does not use an Android runtime permission prompt; it is checked via [`useNetworkStatus.js`](/d:/Aura - Frontend/aura-frontend/src/composables/useNetworkStatus.js) and shown in the offline banner in [`App.vue`](/d:/Aura - Frontend/aura-frontend/src/App.vue)
- Storage permissions are intentionally not declared. APK exports write to the app cache through Capacitor Filesystem and then use Android Share/FileProvider, so broad `READ_EXTERNAL_STORAGE`, `WRITE_EXTERNAL_STORAGE`, or `READ_MEDIA_*` access is not required.
- Background location is intentionally not declared. Attendance geofencing runs only while the user is actively checking in/out.
- Exact alarm permissions are intentionally not declared. Aura uses immediate local notifications for attendance and foreground/return-to-app event alerts instead of exact scheduled alarms.

## Native event notifications

The APK starts a native-only event notification sync from [`eventNotificationSync.js`](/d:/Aura - Frontend/aura-frontend/src/services/eventNotificationSync.js).

- Ongoing events trigger a local notification once per event window.
- Newly loaded events trigger a local notification after the app has already seen the user's existing event list.
- The native app refreshes dashboard events on startup, foreground resume, and a 60-second foreground poll.
- These are local Capacitor notifications. For notifications while the app has never opened or is fully killed, the backend would need server push/FCM.

## Android network config

- `VITE_NATIVE_API_BASE_URL` or `VITE_BACKEND_PROXY_TARGET` controls the backend host used by the APK.
- The generated Capacitor config allows the configured backend hostname plus Railway and ngrok app/dev host patterns.
- The Android network security config permits HTTPS public hosts and cleartext local/intranet hosts for internal APK testing. Prefer HTTPS backend URLs for production builds.

## Install on Android phone

For local testing:

```powershell
adb install -r aura-apk\android\app\build\outputs\apk\debug\app-debug.apk
```

Or copy the APK to the phone and install it manually.

## Rebuild after backend change

If the backend URL changes:

1. Update the env file
2. Run `npm run android:build:debug` again

The Capacitor config will regenerate automatically for the new backend host.
